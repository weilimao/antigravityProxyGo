### v1.1.12 更新日志

本次发布重点对应用进行了全方位的安全加固，在网络连接、敏感凭据保存、API 接口限流校验等方面防御了潜在的安全隐患。同时，优化了代理数据包捕获的内存占用与磁盘 I/O 刷写性能，完成了前端组件的懒加载与 DOM 垃圾回收改造，大幅降低首屏渲染时间与长时间运行下的内存消耗。

#### 1. 全方位安全深度加固 (Security)
- **SSRF 拦截与防 DNS 重绑定 (DNS Rebinding 防御)**：代理连接引入 resolve-then-connect 机制，在对中继请求执行 SSRF 校验后，锁定解析出的第一个安全 IP 进行实际网络建立，规避 TOCTOU 安全漏洞。
- **凭据高安全度加密存储**：采用设备特征熵动态派生密钥，对远程中继密码采用 AES-256-GCM 算法进行高强度加密存储（前缀 `enc:v1:`），并向下兼容旧版本明文凭据，防止 config 文件被直接窃取明文。
- **限制接口登录频次与验证**：中继服务端新增基于 IP 的登录接口限流限制（每 IP 每分钟最高 5 次）；下载 CA 证书接口强制实施 `Bearer Token` 验证，保障证书不被匿名爬取；且客户端在发起下载请求时自适应携带对应的 Token 校验。
- **统一防泄露设计**：统一登录校验失败错误提示为 `invalid credentials`，规避用户名嗅探；在状态展示 IPC 数据中彻底剔除敏感 `token` 信息；另外，限制了用户数据表持久化文件的写权限为 `0600`。

#### 2. 性能与内存专项防御优化 (Performance & I/O)
- **响应体与请求体截断防溢出**：对拦截和转发的数据流进行内存防溢出控制。当请求体/响应体 JSON 大于 50KB 时自动进行截断，规避因为个别超大报文将进程内存撑爆的潜在隐患。
- **捕获数据磁盘 I/O 刷写防抖**：将包捕获模块的磁盘写入改为 3 秒防抖（Debounce）缓冲刷盘机制，避免突发高频请求引发的并发写盘冲突及高 CPU 开销；并将内存捕获条数上限硬限制为 50 条，防止内存无限膨胀。

#### 3. 核心功能及兼容性提升 (Core Features)
- **Token 失效自动重连**：当客户端发现与中继连接因 Token 过期（返回 401 或 407）失效时，会自动调用注册的回调函数，以保存的凭据尝试无感知自动重新登录并刷新 token，保证代理不断连。
- **自适应中继 TLS 降级兼容**：中继服务端支持更平滑的后向兼容，无证书时安全降级回落为 plain HTTP 服务。
- **全局 TLS 证书信任池注册**：客户端在成功下载 CA 证书后，将证书动态注册至 TLS 全局信任池，后续 HTTPS 通信即可基于此 CA 自动完成双向信任验证。

#### 4. 前端首屏与 DOM 垃圾回收优化 (Frontend Optimization)
- **路由组件 v-if 懒加载机制**：主看板内的子页面组件从默认全量挂载修改为 `v-if` 首访懒加载模式，只有页面被初次点击时才实例化挂载，大幅缩减了主页首次渲染的耗时。
- **生命周期解耦初始化**：子页面的 controller 初始化逻辑从 `App.vue` 解耦并迁移到各自页面的 `onMounted` 钩子中，消除了组件未挂载时查找 DOM 报错的隐患。
- **触发日志区域节点回收机制**：对长文本账号自动触发运行日志做了 DOM 垃圾回收处理。子节点上限设定为 100，超出时滚动回收最早的 20 个节点，从根源规避长时间挂机下的网页 DOM 内存泄露及渲染卡顿。
