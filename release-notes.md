### v1.1.0 更新日志

本次更新引入了纯 Go SQLite 本地存储、多模式统计分离及数据增量周期同步等功能，解决了此前无法动态显示使用趋势和获取高并发实时日志的问题：

1. **纯 Go 嵌入式 SQLite 存储 (No CGO)**：
   - 弃用纯内存/JSON 读写，迁移至纯 Go 编写的 SQLite 驱动（`modernc.org/sqlite`），无 CGO 依赖，完美保障 Windows 等桌面环境的跨平台交叉编译和部署。
   - 建立 `request_logs` 结构化存储，全面支持并记录请求的各项 Token 及费用指标（InputCost、OutputCost、CachedCost）。
   - 默认启用 WAL 模式并限制连接池，极大增强了并发写入的稳定度与读性能。

2. **多模式数据统计分离**：
   - 引入远端中继代理模式（Remote Relay Mode），支持配置远程代理设置和处理远程访问请求。
   - 实现了远端（Remote）与本地（Local）统计数据的强隔离，解决不同模式下请求统计冲突混淆的问题。

3. **增量日志自动同步机制**：
   - 抛弃以往“仅在客户端启动时全量同步一次”的设计。
   - 服务端（Relay）提供高效的 `/api/logs/sync` 游标同步接口。
   - 客户端引入每 60 秒执行一次的后台同步协程，利用最新 `server_log_id` 自动请求增量更新并持久化至本地。

4. **大盘使用趋势实时动态更新**：
   - 采用基于 SQLite 本地数据库的聚合查询，替换了旧版的内存计算。
   - 保证了前端“使用趋势”大盘能够伴随每 60 秒的同步自动刷新渲染，彻底实现数据变化的可视化。

### v1.0.8 更新日志

本次更新主要解决了 Windows 环境下，由于系统用户名包含中文或特殊字符导致 Wails CLI 编译报错、以及系统未正确配置 NSIS 环境变量导致打包失败的问题：

1. **新增 Windows 兼容性构建批处理脚本**：
   - 新增 `dev.bat` 和 `build.bat` 用于替代原生 `wails dev` 和 `wails build`。
   - 自动在项目根目录下创建并挂载 `.wails_temp` 作为编译临时目录，彻底规避了 Windows 中文用户名路径导致的 Wails CLI 临时配置文件读写失败（`wails.json` 找不到）及资源文件（`.syso`）清理崩溃 Bug。

2. **自动检测与挂载 NSIS 编译器**：
   - 在 `build.bat` 中加入对 Windows 默认 NSIS 安装目录（`C:\Program Files (x86)\NSIS` 和 `C:\Program Files\NSIS`）的自动检测。
   - 编译时会自动将 NSIS 目录加入到会话 `PATH` 环境变量中，免去开发者手动配置系统全局环境变量和重启电脑的步骤，一键输出安装包。

3. **忽略文件及文档更新**：
   - 将 `.wails_temp` 临时目录加入 `.gitignore`，防止编译器缓存污染 Git 仓库。
   - 更新 `README.md`，增加对于中文 Windows 用户兼容脚本的使用说明。

### v1.0.7 更新日志

本次更新主要针对号池拆分、路由逻辑、会话管理以及额度监控进行了多项功能增强和修复优化，提升了系统的稳定性和多账号调用的灵活性：

1. **Gemini CLI 独立号池拆分**：
   - 实现了对 Gemini CLI 的拆分，支持独立的号池管理，使底层 CLI 调用与核心代理服务的账号管理更加清晰、解耦。

2. **多账号会话绑定与证书注入优化**：
   - 优化并修复了证书注入的相关问题，确保 HTTPS MITM 代理的稳定性。
   - 引入了绑定会话标记，为跨请求的会话一致性提供了底层保障。

3. **路由逻辑与负载均衡优化**：
   - 优化了项目池（Project Pool）的路由逻辑。
   - 修复了代理项目 ID 错误问题。
   - 解决会话 ID 不一致以及粘性路由（Sticky Routing）在复杂网络环境下的负载均衡漂移问题，提升了请求中转效率。

4. **使用趋势与额度自动获取**：
   - 优化了使用趋势指标（Usage Trend Metrics）的数据收集 and 展示。
   - 修复了开机自启路径（Autostart Path）的配置问题。
   - 新增自动获取账户额度（Quota）功能，提升账号使用透明度。
