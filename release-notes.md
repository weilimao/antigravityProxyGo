### v1.1.1 更新日志

本次更新引入了中继代理模式下的套餐（Packages）模板化管理、细粒度多模型配额控制以及前端交互界面升级，解决了多账户使用额度难以批量划分配套及周期性流量限制的痛点：

1. **中继端套餐（Packages）模板化管理**：
   - 新增 `packages.go` 套餐模块，内置 Pro、Pro 5x、Pro 20x 等多套配额模板。
   - 支持对套餐模板进行新增、修改和删除的 CRUD 管理，极大简化了多用户额度的批量配置工作。

2. **细粒度的多模型族配额控制**：
   - 重构了 `UserQuotas` 配额结构。现在对 Gemini 和 Claude 模型族支持独立的固定额度限制（Fixed）和周期额度限制（支持按小时 Hourly、按天 Daily 限制及 Token 数限制）。
   - 支持套餐有效期配置，系统能够根据设定的有效期（月/年/天）自动计算和绑定用户的到期时间。

3. **数据库统计层深度支持**：
   - 扩展了 SQLite 本地存储层，新增按模型族群（Gemini/Claude）查询特定时间跨度内已用 Token 的聚合计算方法，实现了对每小时、每天周期性额度的精准监控。

4. **前端中继管理端交互全方位升级**：
   - 中继配置页集成套餐模版配置表单，支持可视化创建套餐模版。
   - 用户卡片新增“分配套餐”和“自定义配置”高级配额弹窗，支持灵活配置与下发限流策略。
   - 前端控制台实时计算并展示个人配额状态，包括当前已用 Token 数及下一次额度重置的倒计时/时间点。
