### v1.1.7 更新日志

本次发布包含前端自适应布局的精细优化、本地数据库迁移逻辑的重大 Bug 修复、以及中继限额算法的重构：

#### 1. 前端导航栏自适应与体验优化
- **响应式导航栏折叠 (Dropdown)**：支持依据各组件真实占位宽度（非固定 px）动态计算折叠阈值。在窄屏下，自动将导航选项收纳为精致的下拉菜单并高亮当前路由；宽屏空间充足时自动展开。
- **远程连接文字动态隐藏**：当屏幕宽度变窄时，自动在小屏下以 CSS 级联规则强制隐藏连接 IP 长文本（如 `远端: 172.16.x.x`），仅保留云图标与其余控制按钮。完美解决了 Vue 虚拟 DOM 渲染覆写与原生 Wails 控制器之间的状态冲突，确保布局不重叠且连接状态百分百稳定。
- **菜单国际化翻译补全**：补全了“账号池”、“使用详情”、“2FA验证码”以及“抓包分析”的中英文对应翻译，使“中/EN”切换在标准栏与下拉菜单中无缝同步。
- **抓包项显隐联动**：使设置中开关“抓包分析”时大屏和下拉栏的链接能够完全同步隐藏与显示。

#### 2. 本地数据库版本化迁移 (Migrations) 重构
- **引入版本化迁移控制**：在 SQLite 数据库初始化逻辑中加入了迁移版本管理字段（`migration_version`）。
- **修复趋势数据无限累加 Bug (V1)**：修复了原本 V1 迁移脚本在每次软件启动时都会重复运行，导致 `user_hourly_trends` 趋势统计表内的数据重复累计翻倍的严重缺陷。
- **添加污染数据自动恢复 (V2)**：新增 V2 迁移。当检测到低于此版本时，会自动根据权威的 `request_logs` 原始请求日志，一键重新生成并覆盖校正已受污染的 `user_hourly_trends` 历史趋势数据。

#### 3. 中继限额算法重构 (Fixed Window)
- **从滑窗策略切换到固定对齐窗口**：引入 `FixedWindowBounds` 机制，将原本根据最近请求时间向前推算的滑窗逻辑，重构为基于固定锚点时间（本地 `2025-01-06 00:00`）对齐的固定时间窗口限额管理。
- **稳定的重置预期**：对 Gemini 和 Claude 模型的 Hourly 与 Daily 限额使用量以及重置刷新时间（`resetAt`）进行了重构，使用户对配额重置刷新时刻拥有完全可预测的稳定预期。
