### v1.0.7 更新日志

本次更新主要针对号池拆分、路由逻辑、会话管理以及额度监控进行了多项功能增强和修复优化，提升了系统的稳定性和多账号调用的灵活性：

1. **Gemini CLI 独立号池拆分**：
   - 实现了对 Gemini CLI 的拆分，支持独立的号池管理，使底层 CLI 调用与核心代理服务的账号管理更加清晰、解耦。

2. **多账号会话绑定与证书注入优化**：
   - 优化并修复了证书注入的相关问题，确保 HTTPS MITM 代理的稳定性。
   - 引入了绑定会话标记，为跨请求的会话一致性提供了底层保障。

3. **路由逻辑与负载均衡优化**：
   - 优化了项目池（Project Pool）的路由逻辑。
   - 修复了代理项目 ID 错误问题。
   - 解决会话 ID 不一致以及粘性路由（Sticky Routing）在复杂网络环境下的负载均衡漂移问题，提升了请求中转效率。

4. **使用趋势与额度自动获取**：
   - 优化了使用趋势指标（Usage Trend Metrics）的数据收集 and 展示。
   - 修复了开机自启路径（Autostart Path）的配置问题。
   - 新增自动获取账户额度（Quota）功能，提升账号使用透明度。
