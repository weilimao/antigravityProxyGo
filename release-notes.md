### v1.1.5 更新日志

本次更新重点重构了多选账号刷新配额的测试回复机制，并实现了强大的定时与配额触发自动化测试调度模块，同时针对 JSON 响应回显和 SQLite `NULL` 数据加载进行了体验优化与漏洞修复：

1. **配额测试刷新交互全面升级**：
   - 移除原有多选测试下拉框，统一改为轻量级“触发测试刷新”闪电控制按钮。
   - 挂载独立的交互配置 Modal。在触发时提供自定义 Prompt 输入框、测试模型分组复选网格（提供全选、清除、分组样式）。
   - 增加等宽字符实时日志板，提供后台执行日志的滚屏渲染高亮，并增加针对测试成功的账号及模型在底部的结构化结果汇总表格。

2. **响应 JSON 提取与实时日志面板投射展示**：
   - 后端触发模块兼容并拆包官方通道（Antigravity Provider）返回的嵌套 `{"response": ...}` JSON 结构，精准从中提炼出单 token 回复文本（如 `"ok"`）呈现于表格。
   - 优化详情列的交互样式，允许用户直接点击单元格，自动对该行的 JSON 响应或报错信息进行 4 空格换行缩进格式化，并追加输出到进程日志板中进行高亮且清晰的阅读。

3. **自动化测试与定时任务包服务（AutoTrigger Suite）**：
   - 数据库中新增 `auto_trigger_tasks` 结构，提供持久化存储 CRUD 操作，支持自动化包跨会话热重载。
   - 编写后台自动调度器 `Scheduler`，采用 Go 协程和 Ticker 每 10 秒轮询数据库中开启的 `timer` 定时触发任务。
   - 实现 `OnQuotaRefreshed` 事件钩子，并在配额刷新成功后进行异步广播。一旦检测到某个账号配额刷新完毕，便立即触发跟该账号相关联的所有 `quota_refreshed` 任务包，保持最智能、最及时的冷静期重置。

4. **账号展示去重、归属项目 ID 区分与表单项默认值优化**：
   - 定时任务配置表单中的关联账号网格只渲染当前选定通道的账号，官方通道相同邮箱实例自动过滤去重。
   - 云项目项目通道下的邮箱，自动在后缀括号中以 `(项目: projectId)` 显示标注对应项目名，以做清晰区分。
   - 新建任务表单时的账号与模型复选框默认状态从默认勾选变更为默认全不选，方便用户按需点选。

5. **SQLite 读取 Scan NULL 容错修复**：
   - 针对未执行定时或新建的任务，其 `next_trigger_time` 为 `NULL` 进而引发转换至 `string` 报错的缺陷，将扫描目标修改为 `sql.NullString` 结构并增加有效性防护，从根源上消除了列表渲染异常。
