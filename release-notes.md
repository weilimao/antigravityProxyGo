### v1.1.4 更新日志

本次更新引入了单条接口抓包日志的 Markdown 一键导出功能，大幅增强了 OpenAI 新版 Responses 协议与工具调用兼容翻译能力，并修复了网络代理在高压/异常响应下的日志解压显示乱码问题：

1. **单条抓包日志 Markdown 一键导出**：
   - 抓包详情页新增独立的“导出 MD”控制按钮，支持在无数据包选中时动态隐藏。
   - 重构导出逻辑与结构化格式化组件（`packetFormatter.ts`），自动将选中的抓包记录转换为包含基础属性、请求头/请求体、响应头/响应体的标准 Markdown 文档。
   - 联调后端 IPC 保存交互，自动清洗文件名中的非法字符，弹出系统文件保存框，方便开发者归档分析。

2. **OpenAI 新版 Responses 协议与工具转换适配增强**：
   - 深度兼容 OpenAI 新版 Responses API 格式（自动解析异构 `input` 结构体及 `tools` 定义），并将其无缝平滑翻译为标准的内部消息格式。
   - 优化 API 兼容翻译引擎中的角色映射机制，引入智能角色合并（`mergeConsecutiveRoles`），解决连续相同角色在 Gemini API 中触发严格约束报错的问题。
   - 完善工具调用与工具响应的双向映射能力（支持 `tool_calls` 与 `tool` 消息至 Gemini `functionCall` / `functionResponse` 的转换）。
   - 拓展模型映射规则，新增 `gpt-5` 到 `gemini-3-flash-agent` 的自动识别与调度支持，并在 SSE 响应流中完美兼容推送 Responses 格式相关事件。

3. **网络代理 Gzip 响应解压日志记录修复**：
   - 修复代理服务在捕获状态码为 400 及以上且响应报文启用了 Gzip 压缩时，错误日志摘要直接输出二进制乱码的缺陷。
   - 引入动态响应解压模块（`decompressIfNeeded`），自动识别 `Content-Encoding: gzip` 并解压，确保异常调试日志始终清晰直观。

### v1.1.3 更新日志

本次更新引入了全新的中继 API 客户端兼容层、可视化的模型映射配置面板，并对远程连接和冷却提示逻辑进行了进一步的优化与体验提升：

1. **中继大模型映射配置（Model Mapping）**：
   - 前端中继管理面板新增独立的“模型映射”标签页，支持可视化的添加、修改和删除客户端模型与底层实际模型的映射关系。
   - 提供是否对外“暴露”（Expose）给客户端模型列表的开关选项，配置完成后自动热加载，实现极高的灵活性。

2. **OpenAI / Anthropic 客户端 API 兼容层（Compat Layer）**：
   - 新增 `APICompatHandler` 服务，自动适配并将第三方客户端调用 `/v1/models`、`/v1/chat/completions`、`/v1/messages` 的请求完全翻译为 Gemini 协议，完美复用已有的多渠道账号池及分发与重试逻辑。
   - 兼容 SSE（Server-Sent Events）流式响应输出、工具调用（Tool Use / Function Calling）以及思维链签名（Thought Signature）的翻译与双向数据还原，提供如同原生 API 般的流畅体验。

3. **远程中继连接（Remote Mode）管理升级**：
   - 顶部远程连接栏新增“复制 Key”快捷按钮，方便用户快速一键复制本端 30 天有效的中继 API 密钥用于绑定调用。
   - 支持后端同步协议下的 `remoteToken` 参数，并微调了点击“一键刷新”额度时的视觉动画反馈时间（强制至少保持 800ms 旋转），提供平滑的交互感知。

4. **账号冷静期（Cooldown）机制细化**：
   - 优化冷却算法，冷静期状态展示从显示“最大冷却恢复时间”变更为显示“最小冷却恢复时间”（即第一批账号预计恢复的时间），使用户能够更早、更精准地获悉下一次可用额度的到来。

### v1.1.2 更新日志

本次更新对前端架构进行了现代化升级，重构了中继服务安全控制以及管理端的分页逻辑，实现了性能和体验的全面优化：

1. **前端技术栈 Vue 3 + Vite 5 重构**：
   - 引入 Vue 3, Vue-Router 与 Pinia (状态管理)，将 Vite 构建工具升级至 v5。
   - 对传统的单体 `index.html` 页面进行了深度重构与模块化组件拆分，提高了界面的可维护性。
   - 在个人中心/用量统计（`usageDetails`）中引入了前端过滤搜索与分页显示，有效解决了在大批量账号数据时渲染卡顿的问题。

2. **中继服务网络安全防御与热加载**：
   - 在代理内核 and 配置管理器中引入了 SSRF 防御（自动拦截私有和回环网段 IP）、端口放行限制（默认仅限 80/443 通讯）以及域名黑白名单过滤机制。
   - 支持安全配置的随时热加载更新，无需重启服务。
   - 前端新增“中继安全拦截设置”控制面板（包含 SSRF、端口限制与域名白名单配置），支持可视化配置。

3. **中继管理端服务端分页与检索**：
   - 优化了获取中继用户列表的 `relay:get-users` IPC 服务。后端新增内存检索与服务端分页参数，前端重构对接分页接口并适配检索和套餐过滤器。

4. **TOTP 与逻辑解耦优化**：
   - 将原来混合在主入口 `app.go` 中的 TOTP (2FA) IPC 接口逻辑解耦，独立拆分至新文件 `app_totp_ipc.go` 中，进一步强化“高内聚低耦合”架构。
   - 本地请求日志的时间戳展示优化为本地时区可读格式 (`01/02 15:04:05`)。
   - 对接客户端模式下的远程中继，支持在 TLS 隧道中实现本地 IDE 请求的流式转发捕获及包历史记录，不影响本地控制台展示。
