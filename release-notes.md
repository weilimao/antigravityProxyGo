### v1.1.6 更新日志

本次更新重点为代理软件引入了抓包分析菜单的隐藏与本地流量抓包的控制开关，使用户能够自由配置该功能是否对界面和存储生效：

1. **抓包分析设置开关项**：
   - 在“系统设置”的“参数配置”页面中，新增了“抓包分析设置”卡片开关。
   - 默认状态为开启。当用户手动关闭该开关时，顶部导航菜单栏的“抓包分析”菜单项会立即动态隐藏；重新打开时则即时显示。
   - 在关闭开关状态下，如果用户当前正好处于“抓包分析”视图，程序将自动重定向切换至“控制台”视图。

2. **本地接口包捕获与存储拦截控制**：
   - 联动后端数据流。在抓包设置关闭时，后端的包捕获记录器 `PacketCapturer` 将立刻停止对任何上游 API 调用（包括请求头、请求体、响应头、响应体）的捕获与本地持久化存盘（`captured_packets.json`），彻底节省本地磁盘读写并保护接口隐私。
   - 兼容老旧配置文件结构。即使老旧的配置文件中缺失 `enablePacketCapture` 属性，系统反序列化后也将保证其逻辑默认处于开启状态。
