### v1.0.3 更新日志

本次更新重点针对高并发场景进行了底层连接、并发控制及线程安全优化，并丰富了前端请求日志：

1. **高并发优化与网络连接复用**：
   - 彻底移除了每次建连时本地 VPN 端口的同步探测，消除 70ms 建连阻塞。
   - 升级连接池参数（`MaxIdleConns=200`，`MaxIdleConnsPerHost=100`），极大提升长连接复用率。
   - 提升 MitmListener 通道并发缓冲区至 `2048`，并将超时断连改为 3 秒排队等待。

2. **全流程线程安全与锁粒度细化**：
   - 引入 `tokenMu` 局部锁，多线程安全更新与获取 AccessToken，规避全局锁死锁。
   - 粘性会话路由（Session Router）升级为读写分离锁与双重校验锁定，提升高并发读取吞吐量。
   - 统计信息写盘时执行深拷贝后立即释锁，在锁外执行 JSON 反射，防止高并发下因反射锁死主线程。

3. **500 阶梯式原地重试与 Context 感知退避**：
   - 优化 `SERVER_ERROR` 的容错重试策略：遇到服务端错误时，在当前账号原地退避重试，连续失败 3 次后才冷却隔离并执行账号切换（故障转移）。
   - 重试等待采用 Context 敏感的非阻塞 `select`，当客户端或 IDE 主动取消请求时瞬间退出并释放连接。

4. **请求日志增加“耗时（Duration）”字段**：
   - 前端日志表格和详情弹窗新增耗时（Duration）字段，并对秒级与毫秒级进行了美观的自动格式化展示。
   - 补全了对应的中英文 i18n 国际化翻译。

5. **系统操作增强**：
   - Go 后端 IPC 通信支持 `settings:open-folder` 指令，支持直接调起系统默认资源管理器打开指定目录。

### v1.0.2 更新日志

本次更新集中优化了前后端的内存抖动、渲染性能，并校正了真实的物理内存占用，新增了 CPU 占用率监控：

1. **Windows 内存真占用校正**：
   - 升级内存抓取机制为 `PROCESS_MEMORY_COUNTERS_EX`，采用 `PrivateUsage`（私有专用提交大小）代替工作集大小。
   - 彻底排除了 Wails 与多个 WebView2 子进程（渲染、GPU、网络等进程）之间共享系统 DLL 导致的 7 倍内存多重累加，使界面“系统物理内存”显示真实的专属物理开销，与任务管理器相匹配。

2. **新增 CPU 占用率监控**：
   - 引入 Windows 进程时间（GetProcessTimes）差量计算器，高精度跟踪主进程和所有 WebView2 子进程的 CPU 核心时间消耗。
   - 结合多核处理器数量（`NumCPU`）智能算得并展示整套应用的实时 CPU 综合占用率。

3. **前端 DOM 节点复用与渲染节流（优化 95% DOM 损耗）**：
   - **账号卡片增量更新**：重构 `renderAccounts`，通过 `data-account-id` 精准识别与原地复用保留卡片，通过 DOM 排序原地调整位置。仅在冷静状态、积分数额、启用开关发生变动时对局部 DOM 节点执行增量 Patch，消灭了每次状态刷新都全量销毁重建几百个 DOM 节点的 GC 开销。
   - **趋势图 SVG 复用**：折线图网格线（5条）及 X/Y 轴的 HTML Label 节点改为初始化一次并按需更新文本或坐标属性，淘汰了每次都使用 `innerHTML = ''` 的暴力重建。
   - **控制台滚动与批量裁剪**：控制台滚动加入 `requestAnimationFrame` 防抖保护，按帧合并多次滚动，且折叠时完全不执行滚动；将日志超限裁剪改为每满 150 条一次性批量清除 30 条。
   - **脏数据渲染拦截**：通过生成包含 stats/trends/requests/usage 的签名，拦截无实质数据变化的 `stats-updated` 定期心跳推送，实现挂机空闲时零重绘损耗。

4. **Go 后端性能优化**：
   - 正则表达式优化：将 high-frequency 的 `regexp.MustCompile` 统一提升为包级别的全局常量，消除了每次代理请求时无谓的正则编译和短命 regexp 垃圾分配。
   - 证书缓存保护：对 `certCache` 加入了最大 50 个的上限容量保护，防止极端情况下的无限制内存泄露。

### v1.0.1 更新日志

本次更新对桌面客户端进行了内存优化与问题修复：

1. **限制 WebView2 虚拟机的 V8 堆内存**：
   - 增加启动参数 `--js-flags="--max-old-space-size=128"`，将渲染进程的 JS 内存堆硬性限制在 128MB，强力触发 V8 垃圾回收，防止长期运行或高频数据交互导致内存不断膨胀。
   - 禁用 GPU 的着色器程序缓存（`--disable-gpu-program-caches`等）并定时主动修剪 GPU 指令缓冲区，释放无用的显存和渲染占位。

2. **高频 IPC 数据传输载荷精简（优化 90% 数据量）**：
   - 引入简易版载荷函数，对于高频触发的报文监测和状态更新，剔除包含 720 条历史时段小时趋势图的 `trends` 数组，使每次高频传输的 IPC 载荷体积缩小了 90% 以上。
   - 增加低频更新机制，将趋势折线图更新降低为每 60 秒触发一次，在保证核心报文与数值卡片实时显示的前提下，极大降低了渲染进程和 Go 进程的 JSON 解析损耗。

3. **前端渲染性能优化与条件重绘**：
   - 引入页面激活状态机制，只有当处于对应页面（如主页、账号用量页）时才执行对应的折线图重绘和复杂的 HTML 列表重构。
   - 当应用处于后台或处于“设置”、“报文”等不包含相关列表的页面时，完全跳过大开销的 DOM 重建与渲染，彻底消除了由频繁后台重绘导致的 DOM 垃圾和 detached 节点泄漏。

4. **Tailwind CSS 静态化（AOT 编译优化）**：
   - 彻底移除了运行时 418KB 的 `tailwind.min.js` 实时编译器，改为在构建期（AOT）静态提取编译为最小化样式文件，减少了运行时的 CSS 解析计算压力。

5. **内存指标概念校正**：
   - 将主界面的“实际内存”重命名为“系统物理内存”（反映整个进程树真实物理 Working Set 内存开销）；
   - 将“预留内存”重命名为“Go 堆内存”（精准显示 Go 虚拟机实际占用的 Heap 资源），与系统任务管理器中的实际多进程开销对齐。

6. **修复自动更新重启失效问题**：
   - **修复了下载更新后点击“立即重启”按钮无响应的漏洞**（补全了 Go 后端缺少的 `"app:install-update"` 消息通信通道绑定，使之能顺利启动安装包并退出当前程序）。
