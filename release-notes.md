### v1.1.8 更新日志

本次发布重点解决了 Windows 平台 Clash TUN 虚拟网卡代理冲突、中继思考过程特征签名清洗，并加入了专属 API Key 限额管理、专属 SOCKS5 代理和实时的网络连接状态监控看板：

#### 1. API Key 限额与使用量管理
- **API Key 级别配额限制**：在 API Key 配置中新增对 Gemini 和 Claude Token 配额的单独限额。
- **已用 Token 统计记录**：支持自动记录和统计每个 API Key 消耗的 Token 数量，便于精细化额度管理与鉴权。

#### 2. 工具调用与思考特征签名（Thought Signature）清洗
- **对话接口工具调用重构**：优化并增强了 OpenAI/Anthropic 兼容接口中 tool calls 的双向转换逻辑。
- **思考签名防泄漏**：彻底禁用将中继思考签名暴露在明文返回中的行为，引入强力正则过滤清洗，移除所有残留的 `<!--thought_signature:...-->` 注释，保证纯文本响应的整洁与安全。

#### 3. 自动化任务定时触发逻辑重构
- **到达重置时间触发**：从“依赖配额刷新事件触发”更改为“主动轮询检测账号是否到达配额重置时间 (`CooldownUntil`) 触发”，解决原来部分账号触发时机不准确的问题。
- **UI 与文案同步优化**：前端“自动触发配置”中的选项文案同步变更为“到达配额重置时间触发”。

#### 4. Windows 虚拟网卡 (Clash TUN) 冲突修复与本地 Fallback 降级
- **自适应回环降级**：当 IE 系统代理为空时，后台探测协程每 `15 秒` 自动扫描本地运行中的常见 VPN 端口（Clash, v2rayN 等），协议特征分析通过后强制以 127.0.0.1 回环通信，彻底绕过拦截驱动的握手卡死。
- **自定义 Fallback 端口**：支持在界面中自定义扩展多个扫描端口。

#### 5. 专属 SOCKS5 代理配置（绕过系统代理）
- **独立代理路径**：增加专属 SOCKS5 代理开关与地址配置，启用后所有外部流量强行只通过该 SOCKS5 中转，彻底绕过系统代理的获取与影响。

#### 6. 连接池假死修复 (DisableKeepAlives)
- **禁用长连接复用**：针对代理软件可能随时重启或切换节点的问题，底层对所有外部出站请求强制配置 `DisableKeepAlives: true`，拒绝死连接残留并强迫每次请求都动态调用最新代理重拨，彻底解决了网络切换时连接卡死无响应的问题。

#### 7. 实时网络健康看板与出站路由日志监控
- **新页签监控看板**：系统设置中新增了**“网络监控”**卡片。通过 3秒自动轮询推送展现当前 Fallback 探测值、专属代理状态。
- **出站路由跟踪日志**：在 `DialContext` 建连底层加装出站追踪，实时记录最近 100 条拨号的目标、所选代理、连接耗时以及成功/失败结果，帮助看清每一次请求的实际走向。

#### 8. 架构解耦与防腐
- **IPC 桥梁解耦**：新增 `app_settings_ipc.go` 接收端，将所有设置类 IPC 动作抽离，遵循严格的单文件行数限行约束，防止 `app.go` 膨胀。
